## рҹҡҖ Quick Start

### Prerequisites
- Node.js 20+ and pnpm
- Python 3.12+
- Docker & Docker Compose
- OpenAI API key
- Pinecone account (free tier works)

### 1. Clone & Setup

```bash
git clone https://github.com/Maneesh-k/AI-Resume-Screening-System.git
cd AI-Resume-Screening-System

# Copy environment variables
cp .env.example .env
```

### 2. Configure Environment

Edit `.env` and set the required values:

```env
# Required for core AI features
OPENAI_API_KEY=sk-...
PINECONE_API_KEY=pcsk_...

# Required for file storage (use LocalStack for local dev)
AWS_ACCESS_KEY_ID=your-key
AWS_SECRET_ACCESS_KEY=your-secret
AWS_S3_BUCKET_NAME=ai-hiring-copilot-resumes

# Auto-configured for Docker Compose
DATABASE_URL=postgresql+asyncpg://postgres:postgres@localhost:5432/hiring_copilot
REDIS_URL=redis://localhost:6379/0
```

### 3. Start with Docker Compose (Recommended)

```bash
# Start everything: API + Worker + Web + Postgres + Redis
make up

# Or manually:
docker-compose up --build
```

Access:
- **Frontend**: http://localhost:3000
- **API**: http://localhost:8000
- **API Docs (Swagger)**: http://localhost:8000/docs

### 4. Local Development (Without Docker)

```bash
# Terminal 1 вҖ” Start infrastructure
make up-db          # Starts Postgres + Redis only

# Terminal 2 вҖ” Backend
cd apps/api
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"
alembic upgrade head      # Run DB migrations
uvicorn src.main:app --reload --port 8000

# Terminal 3 вҖ” Frontend
cd apps/web
pnpm install
pnpm dev                  # Starts on :3000
```

### 5. Seed Demo Users

Run the seed script to create demo accounts:

```bash
cd apps/api
source .venv/bin/activate
python scripts/seed.py
```

## рҹӨ– AI Features Deep Dive

### Scoring Engine (5 Dimensions)
```
Skill Match        35% вҖ” Required + preferred skills overlap
Experience Match   30% вҖ” Years + role relevance
Domain Match       20% вҖ” Industry/domain alignment
Education Match    10% вҖ” Degree relevance
Certification Match 5% вҖ” Relevant certifications

Overall = ОЈ (score Г— weight)  [0вҖ“100]
```

### Prompt Engineering
All prompts are in `apps/api/src/infrastructure/ai/prompts/`:
- `resume_parse.txt` вҖ” Extracts structured JSON from raw resume text
- `candidate_score.txt` вҖ” Evaluates candidate against JD (5 dimensions)
- `summary_generate.txt` вҖ” Generates professional 3-5 sentence summary
- `question_generate.txt` вҖ” Creates 8 targeted interview questions

### Model Configuration
```python
LLM:       GPT-4o, temperature=0.2, response_format=json_object
Embedding: text-embedding-3-large, dimensions=3072

Retry:     Exponential backoff (1sвҶ’2sвҶ’4s, max 3 attempts)
```

## рҹ”’ Security

- **Authorization**: RBAC вҖ” Admin, Recruiter, Hiring Manager with permission guards per endpoint
- **Data**: TLS in transit, AES-256 at rest (RDS + S3), all secrets in AWS Secrets Manager
- **PII Protection**: Candidate names/emails/phones never logged вҖ” only entity IDs
- **File Validation**: MIME type + magic bytes validation (not just extension)
- **Rate Limiting**: Redis-based per-user limits (AI: 60/min, Upload: 10/min, Chat: 20/min)
- **S3 Access**: Pre-signed URLs only (TTL: 1 hour) вҖ” no public bucket access
